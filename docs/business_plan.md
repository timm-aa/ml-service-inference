# Краткий бизнес-план ML Prediction Service

## УТП (уникальное торговое предложение)

Сервис позволяет командам и частным пользователям **загружать собственные модели scikit-learn**, выполнять **асинхронные предсказания по REST API** и платить только за успешные запросы во **внутренней валюте (кредитах)**. Встроены **уровни лояльности**, автоматически снижающие цену при высокой активности, что стимулирует регулярное использование без ручных промокодов.

## Целевая аудитория

- Внутренние продуктовые команды, которым нужен контролируемый inference без деплоя своего GPU-кластера.
- Образовательные проекты и студенческие работы с понятной экономикой «кредиты за успешный вызов».

## Источники дохода

1. **Продажа кредитов** через интеграцию с платёжным провайдером (в проекте — заглушка callback).
2. **Подписка B2B** (не реализована в MVP): фиксированный пакет кредитов и SLA по времени ответа очереди.

## Упрощённая финансовая модель (пример)

| Параметр                     | Значение |
|-----------------------------|----------|
| Базовая цена предсказания    | 10 кредитов |
| Средняя скидка по лояльности | 5–10% от базы |
| Себестоимость инфраструктуры | условные фиксированные расходы на VPS/GPU |
| Точка безубыточности (условно) | при N успешных предсказаниях в месяц при заданном тарифе кредита |

Точные цифры зависят от стоимости хостинга и конверсии «рубль → кредит». Метрики из Grafana/Prometheus и журнал транзакций позволяют оценить фактический **ARPU** и **расход кредитов на пользователя**.

## Риски

- Злоупотребление API — смягчается аутентификацией JWT и учётом баланса.
- Долгие задачи — смягчается асинхронной очередью и мониторингом длины очереди Redis/Celery.
